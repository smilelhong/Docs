# Mustache

我们可以向下面代码块中的模式来插入数据

<div id="app">
  <h2>Hello {{ name }}</h2>  //直接插入到标签当中
  <h2>{{ firstname }} {{ lastname }}</h2> //使用两个Mustache进行拼接
  <h2>{{ counter * 2 }}</h2> //同样可以是一个表达式
</div>

<script src="vue.js"></script>
<script>
let app = new Vue({
  el: '#app',

# v-once

  • 该指令后面不需要跟任何表达式(比如之前的v-for后面是由跟表达式的)
  • 该指令表示元素和组件(组件后面才会学习)只渲染一次，不会随着数据的改变而改变。
<div id="app">
  <h2 v-once>{{ message }}</h2>  
</div>

<script src="vue.js"></script>
<script>
let app = new Vue({
  el: '#app',
  data: {
    message: 'Vue.js',
  }
})
</script>

当message的值改变时，页面不会冲渲染。

# v-html

  • 某些情况下，我们从服务器请求到的数据本身就是一个HTML代码，这个时候需要使用v-html
    • 如果我们直接通过{{}}来输出，会将HTML代码也一起输出。
    • 但是我们可能希望的是按照HTML格式进行解析，并且显示对应的内容。
    • v-html指令后面往往会跟上一个string类型，会将string的html解析出来并且进行渲染
<div id="app">
  <h2>{{ link }}</h2>
  <h2 v-html="link"></h2> //
</div>

<script src="vue.js"></script>
<script>
let app = new Vue({
  el: '#app',
  data: {
    link: '<a href="www.baidu.com">百度一下</a>',
  }
})
</script>

# v-text

  • v-text作用和Mustache比较相似：都是用于将数据显示在界面中
  • v-text通常情况下，接受一个string类型
<div id="app">
  <h2 v-text="message"></h2>  //该方式不如mustache灵活
  <h2>{{ message }}</h2>  
</div>

<script src="vue.js"></script>

let app = new Vue({
  el: '#app',

  }
})
</script>

# v-pre

  • v-pre用于跳过这个元素和它子元素的编译过程，用于显示原本的Mustache语法。
<div id="app">
  <h2>{{ message }}</h2> 
  <h2 v-pre>{{ message }}</h2>  //直接显示{{ message }}
</div>

<script src="vue.js"></script>
<script>

</script>
  • 第一个h2元素中的内容会被编译解析出来对应的内容
  • 第二个h2元素中会直接显示

  • 在某些情况下，我们浏览器可能会直接显然出未编译的Mustache标签。

# 事件监听

# v-on介绍

  • 在前端开发过程中，我们经常和用于交互。
    • 这个时候，我们就必须监听用户发生的事件，比如点击、拖拽、键盘事件等
    • 在Vue中我们通过v-on指令来进行监听
  • 作用：绑定事件监听器
  • 缩写：@
  • 预期：Function | Inline Sstatement | Object
  • 参数：event

# v-on基础

  • 我们用一个监听按钮的点击事件，来简单看看v-on的使用
    • 下面的代码中，我们使用了v-on:click="counter++”
    • 另外，我们可以将事件指向一个在methods中定义的函数
<div id="app">
  <h2>点击次数：{{ counter }}</h2> 
  <button v-on:click="counter++">按钮1</button>
   <button v-on:click="btnClick">按钮2</button>
   <button @click="btnClick">按钮2</button> //语法糖
</div>

<script src="vue.js"></script>
<script>
let app = new Vue({
  el: '#app',
  data: {
    counter: 0
  },
  methods: {
    btnClick() {
      this.counter++
    }
  }
})
</script>

# v-on参数

  • 当通过methods中定义方法，以供@click调用时，需要注意参数问题：

情况一：如果该方法不需要额外参数，那么方法后的()可以不添加。

  • 但是注意：如果方法本身中有一个参数，那么会默认将原生事件event参数传递进去

情况二：如果需要同时传入某个参数，同时需要event时，可以通过$event传入事件。

<div id="app">
  <h2>点击次数：{{ counter }}</h2> 
  <button @click="btnClick">按钮1</button> 
  <button @click="btnClickTen(10, $event)">按钮2</button>
</div>

<script src="vue.js"></script>
<script>
let app = new Vue({
  el: '#app',
  data: {
    counter: 0
  },
  methods: {
    btnClick(event) {
      console.log(event)
      this.counter++
    },
    btnClickTen(count, event) {
      this.counter +=10
      console.log(event)
    },
  }
})
</script>

# v-on修饰符

  • 在某些情况下，我们拿到event的目的可能是进行一些事件处理，Vue提供了以下的修饰符来帮助我们处理一些事件。
    • .stop - 调用 event.stopPropagation()
    • .prevent - 调用 event.preventDefault()
    • .{keyCode | keyAlias} - 只当事件是从特定键触发时才触发回调
    • .native - 监听组件根元素的原生事件。
    • .once - 只触发一次回调。
//停止冒泡
<button>@click.stop="doThis"</button>

//阻止默认行为
<button>@click.prevent="doThis"</button>

//组织默认行为，没有表达式
<form @submit.prevent></form>

//串联修饰符
<button>@click.stop.prevent="doThis"</button>

//键修饰符，键别名(keyup、keydown指按键下去谈起，一般用up)
<input @keyup.enter="onEnter">

//键修饰符，键代码
<input @keyup.13="onEnter">

//点击回调只会触发一次
<button>@click.once="doThis"</button>

# 条件与循环

# v-if、v-else-if、v-else

  • 这三个指令和JS里面的听见语句if、else、else if类似
  • Vue的条件指令可以根据表达式的值在DOM中渲染或销毁元素或组件

v-if的原理

  • v-if 后面的条件为false时，对应的元素以及其子元素是不会渲染的
  • 也就是根本不会有对应的标签出现在DOM中
<div ip="app">
  <div v-if="score>=90">优秀</div>
  <div v-else-if="score>=80">良好</div>
  <div v-else-if="score>=60">及格</div>
  <div v-else>不及格</div>
</div>

<script src="vue.js"></script>
<script>
  let app = new Vue({
    el: '#app',
    data: {
      score: 98
    }
  })
</script>

WARNING

小问题：如果我们在有输入内容的情况下，切换了类型，我们会发现文字依然显示之前的输入的内容，后续补充。。。。。。

# v-show

  • v-show的用法和v-if很相似，也用于决定一个元素是否渲染

v-if和v-show的区别

  • v-if当条件为false时，压根不会有对应的元素在DOM中
  • v-show当条件为false时，仅仅是将元素的display属性设置为none而已
<div ip="app">
  <h2 v-show="isShow">是否显示</h2>
</div>

<script src="vue.js"></script>
<script>
  let app = new Vue({
    el: '#app',
    data: {
      isShow: true
    }
  })
</script>
  • 当在开发中如何选择v-show和v-if？
    • 当需要在显示与隐藏之间切片很频繁时，使用v-show
    • 当只有一次切换时，通过使用v-if

# v-for

# v-for遍历数组

  • 当我们有一组数据需要渲染时，我们可以使用v-for，在遍历过程主要有两种情况：
    • 在遍历过程不需要索引值
      • v-for="movie in movies"
      • 依次从movies中取出movie，并且在元素的内容中，我们可以使用Mustache语法，来使用movie
    • 在遍历过程中需要索引值
      • 语法格式：v-for=(item, index) in items
      • 其中的index就代表了取出的item在原数组的索引值
<div ip="app">
  <h2 v-for="item in movies">{{ item }}</h2>
  <h2>-------------分割线--------------</h2>
  <h2 v-for="（movie, index) in movies">{{ index }}.{{ movie }}</h2>
</div>

<script src="vue.js"></script>
<script>
  let app = new Vue({
    el: '#app',
    data: {
      movies: ['加勒比号盗1','加勒比号盗2','加勒比号盗3','加勒比号盗4']
    }
  })
</script>

# v-for遍历对象

<div ip="app">

    {{ value }}-{{ key }}-{{ index }}
    </li>
  </ul>

# 组件key属性

  • 官方推荐我们在使用v-for时，给对应的元素或组件添加上一个:key属性

TIP

当某一层有很多相同的节点时，也就是列表节点时，我们希望插入一个新的节点。例如：我们希望可以在B和C之间加一个F，Diff算法默认执行起来是这样的，即把C更新成F，D更新成C，E更新成D，最后再插入E，这样更新效率非常的低。

  • 所以我们需要使用key来给每个节点做一个唯一标识
    • Diff算法就可以正确的识别此节点
    • 找到正确的位置区插入新的节点
  • 所以一句话，key的作用主要是为了高效的更新虚拟DOM

# 监测数组更新

  • 因为Vue是响应式的，所以当数据发生变化时，Vue会自动检测数据变化，视图会发生对应的更新
  • Vue中包含了一组观察数组编译的方法，使用它们改变数组也会触发视图的更新
<div ip="app">
  <h2>{{ message }}<h2>
</div>

<script src="vue.js"></script>
<script>
let app = new Vue({
    el: '#app',
    data: {
      message: 'hello vue'
    },
    methods: {
    updateData() {
      //1.push方法
      this.names.push('lh','why');

      //2.pop方法，取出最后一个
      this.names.pop();

      //3.unshift方法，左边插入
      this.names.unshift('why', 'lh');

      //4.shift方法，左边移除
      this.names.shift()

      //5.splice方法
      //传递一个参数（index），将对应index，以及后面的所有数据删除
      this.names.splice(2)

      //6.sort()排序函数
      this.names.sort()

      //7.reverse反转函数
      this.names.reverse()

      //不会响应式改变
      //this.names[1] = 'lh'
      //可以通过以下方法
      //方法1:
      this.names.splice(1, 1, 'lh')
      //方法2:
      Vue.set(this.names, 1, 'lh')
    }
}
  })
</script>

# 表单绑定

# v-model基本原理

表单控件在实际开发中是非常常见的。特别是对于用户信息的提交，需要大量的表单，Vue中使用v-model指令来实现表单元素和数据的双向绑定。

  • 举例：
    • 当我们在输入框输入内容时，因为input中的v-model绑定了message，所以会实时将输入的内容传递给message，message发生改变。
    • 当message发生改变时，因为上面我们使用Mustache语法，将message的值插入到DOM中，所以DOM会发生响应的改变。
    • 所以，通过v-model实现了双向的绑定。
<textarea v-model="message"></textarea>
<input v-model="message">

TIP

原理：v-model其实是一个语法糖，它的背后本质上是包含两个操作：

​ 1、v-bind绑定一个value属性

​ 2.v-on指令给当前元素绑定input事件

<input type="text" v-model="message">

//等同于

<input type="text"
       v-bind:value="message"
       v-on:input="message = $event.target.value">
<h2>{{ message }}</h2>

# v-model:checkbox

  • 复选框分为两种情况：单个勾选框和多个勾选框
  • 单个勾选框：
    • v-model即为布尔值
    • 此时input的value并不影响v-model的值
  • 多个复选框：
    • 当是多个复选框时，因为可以选中多个，所以对应的data中属性是一个数组
    • 当选中某一个时，就会将input的value添加到数组中
<div ip="app">
	//单个复选框
  <label for="check">
  	<input type="checkbox" v-model="checked" id="check">同意协议
  </label>
  <p>是否选中: {{ checked }}</p>
  
  //多个复选框
  <label for="check">
  	<input type="checkbox" v-model="habbies" value="篮球">同意协议
  </label>
  <label for="check">
  	<input type="checkbox" v-model="habbies" value="足球">同意协议
  </label>
  <label for="check">
  	<input type="checkbox" v-model="habbies" value="网球">同意协议
  </label>
  <p>您选中的爱好: {{ hobbies }}</p>
</div>

<script src="vue.js"></script>
<script>
  let app = new Vue({
    el: '#app',
    data: {
			checked: false,
      hobbies: []
    }
  })
</script>

# v-model:select

  • 和checkbox一样，select也分单选和多选两种情况
  • 单选：只能选中一个值:
    • v-model绑定的是一个值
    • 当我们选中option中的一个时，会将它对应的value赋值到mySelect中
  • 多选：可以选中多个值
    • v-model绑定的是一个数组
    • 当选中多个值时，就会将选中的option对应的value添加到数组mySelects中
<div ip="app">
	//选择一个值
  <select v-model="myselect">
    <option value="apple">苹果</option>
    <option value="orange">橘子</option>
    <option value="banana">香蕉</option>
  </select>
  <p>您最喜欢的水果：{{ myselect }}</p>
  
  //选择多个值
    <select v-model="myselects">
    <option value="apple">苹果</option>
    <option value="orange">橘子</option>
    <option value="banana">香蕉</option>
  </select>
  <p>您喜欢的水果：{{ myselects }}</p>
</div>

<script src="vue.js"></script>
<script>
  let app = new Vue({
    el: '#app',
    data: {
			myselect: 'apple',
      myselects: []
    }
  })
</script>

# v-model修饰符

# .lazy:

  • 默认情况下，v-model默认是在input事件中同步输入框的数据的
  • 也就是说，一旦有数据发生改变对应的data中的数据就会自动发生改变
  • lazy修饰符可以让数据在失去焦点或者回车时才会更新
<!-- 在“change”时而非“input”时更新 -->
<input v-model.lazy="msg" >

# .number：

  • 默认情况下，在输入框中无论我们输入的是字母还是数字，都会被当做字符串类型进行处理
  • 但是如果我们希望处理的是数字类型，那么最好直接将内容当做数字处理
  • number修饰符可以让在输入框中输入的内容自动转成数字类型
<input v-model.number="age" type="number">

# .trim：

  • 如果输入的内容首尾有很多空格，通常我们希望将其去除
  • trim修饰符可以过滤内容左右两边的空格
<input v-model.trim="msg">
